# Factory-specific guidance for skills

## Drive interactive skills as one-area-at-a-time conversations

When a Factory skill involves a user (capturing briefs, defining behaviors, designing approaches, planning work), drive the conversation in small pieces. Each area or decision gets its own turn. Don't produce a document and ask for approval.

This applies through the entire conversation — a common failure mode is starting with small pieces and dumping everything remaining at the end. One question at a time. Let each answer land before moving on.

This pattern fits Factory's brief → behaviors → approach → plan lifecycle. It doesn't fit autonomous skills like reviewers or code processors.
